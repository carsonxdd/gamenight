# Caplan Game Night — Pages Spec (Updated)

## Landing Page (Public / Home)

The landing page already exists with a hero (title, subtitle, Join Now button, Schedule button) and highlights below. We're restructuring the flow and adding new sections.

### Page Flow (top to bottom)

**1. Hero (already exists — keep as-is)**
- Title: "Caplan Game Night"
- Subtitle
- Join Now button (CTA)
- Schedule button

**2. Short Introduction**
- A brief 2-3 sentence section explaining what this is: a community for organizing game nights with friends. Pick games, find a time that works, show up and play.
- Keep it scannable. No paragraphs. This is for the friend-of-a-friend who just got sent a link and needs to understand the site in 10 seconds.

**3. Social Proof / Activity**
- Live stats showing the community is active. Examples:
  - "X members · Y events hosted · Z games available"
  - Or show the next upcoming event name and date
- Keep it to one line or a small row of stat cards. This builds trust — the site isn't dead, people actually use it.

**4. Members Carousel**
- A horizontal auto-scrolling ticker of member cards
- Shows members who have signed up, scrolling continuously left (like a marquee)
- Each card in the carousel is a compact version of the full member card: avatar, display name, and a few game tags. No ranks, no socials — keep it small and visual.
- The ticker should pause on hover so people can read a card
- Infinite loop — when it reaches the end, it seamlessly wraps back to the start
- Include a "View All Members →" link that goes to the full Members page
- If there aren't enough members to fill the strip yet, either duplicate them to fill the space or hide the section entirely until there are enough (say 5+)

**5. Highlight Cards (2-3)**
- Show 2-3 of the most recent or featured highlights from past events
- These are the same highlights that currently live on the main page but trimmed down to just a few
- Each card is clickable and links to the full Highlights page
- Include a "See All Highlights →" link at the end
- Highlights get their own dedicated page (see Highlights Page below)

**6. Footer**
- Discord invite link (prominent)
- GitHub icon (small, subtle)
- carsoncaplan.com link — COMMENTED OUT in code with note: `<!-- TODO: Uncomment when carsoncaplan.com is live -->`
- Basic nav links: About (if logged in, otherwise skip), Schedule, Members
- Copyright / year

## Highlights Page (New, Public)

A dedicated page for all highlights from past events. This is where the existing highlights content moves to.

### Layout
- Grid or vertical feed of highlight cards
- Each card shows: event name, date, a short description or clip/screenshot, and who was involved
- Sorted by most recent first
- This page is public — anyone can browse it to see what game nights look like

### How highlights get here
- For MVP, highlights can be manually added (admin/organizer adds them)
- Future: players could submit clips or screenshots that get approved

## Members Page (Authenticated)

A browsable directory of everyone in the community.

### Layout

Responsive card grid:
- Desktop: 3 columns
- Tablet: 2 columns
- Mobile: 1 column

### Full Member Card

Each card is a "player card" — more detailed than the carousel version. Contents top to bottom:

1. **Avatar** — Discord profile picture, make it prominent
2. **Display name** — their chosen name (future: team tag prepended like `[AZ] Carson`)
3. **Games** — small tags/icons for their selected games. Show top 5-6, then "+X more" if they have more
4. **Ranks** — if they've added ranks, show as subtle badges under the game tags (e.g., "Val: Gold", "League: Plat"). Only show what they've filled in.
5. **Social links** — small icon buttons, only rendered for socials they've actually linked:
   - Discord (always shown)
   - Twitter / X
   - Twitch
   - YouTube
   - Custom link

### Sorting & Filtering (nice to have, not MVP-critical)
- Filter by game: "Show me everyone who plays Valorant"
- Search by name
- Sort by recently active or alphabetical

## Future: Teams (NOT MVP — Roadmap Only)

When teams are built, they layer onto the members page:

- Users can create or join teams
- Teams have a name and a short tag (e.g., `AZ`, `YOLO`, `GG`)
- Team tag displays before member name on their card: `[AZ] Carson`
- Users can be on multiple teams
- Members page gets a "view by team" filter
- No separate teams page needed — just a filtered view of members

Keep the member card name display flexible enough to prepend a tag later. No code needed for this now.
